## 遇到的问题和解决

1. 前端需要统一处理接口返回，但不同接口最初返回格式不一致。解决方式是抽出 `Result<T>`，所有 controller 都统一返回成功或失败结构。
2. 用户身份不能依赖前端直接传入的用户 ID，否则容易被伪造。解决方式是登录后签发 Token，请求时由拦截器解析 Token 并写入当前用户 ID。
3. 数据库字段不断增加，已有本地库可能缺少新字段。解决方式是补充 `schema.sql`，并增加启动迁移逻辑，对旧表缺失字段做兼容处理。
4. 团队加入、邀请和聊天涉及多张表，容易出现重复数据。解决方式是在业务层校验状态，并在数据库层增加唯一约束。
